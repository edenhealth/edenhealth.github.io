start!

(start! queue)

Starts an SQS consumer. Extend (defmethod) the process-event-message with the keywordized event name to add new features.

Messages should be in the format: {‘event’: event_name, ‘data’: map_of_job_data}

They must use the async-job-message-group-id for now.
