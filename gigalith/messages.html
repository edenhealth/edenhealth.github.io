Messages

The SQS Worker reads messages off an SQS queue for async processing of jobs. Jobs are retried 10 times before being put in a deadletter queue for reprocessing later.

Message Format

Event messages should be JSON strings with two top-level keys: event and data. The event value is used for the process-message defmulti. It is keywordized and determines which defmethod to use for processing the message. The data value should be a map including all information needed to process the job.

Messages must be sent with a message-group-id for deduplication. The message-group-id is “eden-asyn-job”.

Example message:

{
  "event": "hint.new-eligible",
  "data": {
    "hint-patient-id": "1dcfdabf-2e99-429f-8aa3-fa777dae2068"
  }
}

SQS Configuration

The worker is capable of creating and configuring the message queue on starting. If manually creating the queue, ensure the queue is a FIFO queue and that it has a deadletter queue to dump messages that fail to process.
