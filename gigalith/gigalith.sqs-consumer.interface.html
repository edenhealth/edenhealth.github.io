gigalith.sqs-consumer.interface

SQS Consumer brick

Instantiate an sqs consumer reading and processing messages from an sqs queue.

To mount:

(declare consumer)

(mount/defstate consumer
  :start! (start! <queue-name> <handle-fn> <dev?>)
  :stop!  (stop! consumer))

handle-fn should take an sqs/received-message-spec for processing. It must sqs/ack successful messages and sqs/nack unsuccessful ones.

Event structure:

{
  :event :string
  :data  :map
}

consumer-spec

handle-fn-spec

[:=> [:cat [:map [:id :string] [:receipt-handle :string] [:source-queue :string] [:queue-name :string] [:attributes :map] [:message-attributes :map] [:body :string] [:body-md5 :string]]] :any]

start!

(start! queue handle-fn)(start! queue handle-fn localstack-host)

Starts an SQS consumer for queue using handle-fn for processing messages.

Messages should be in the format:

{
  :event :string
  :data  :map
}

Arg Schemas:

[(queue :- :string)
 (handle-fn :- handle-fn-spec)]
[(queue :- :string)
 (handle-fn :- handle-fn-spec)
 (localstack-host :- [:or :nil :string])]

Returns:

[:map [:done-channel :any]
      [:message-channel :any]]
