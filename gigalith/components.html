Components (We Call Them Bricks)

Components are essentially libries for bases to use to perform actions relating to the system. We have different types of components:

  • Mounted components - These need to be mounted to be used. Things like database connections are mounted.
  • Helper components - These may need to be initialized, but generally just need to be used.
  • Domain components - These are where most buisiness logic lives.

Mounted Components

Mounted components will have start! and stop! methods. To mount the component:

(ns example
  (:require
    [mount.core :as mount]
    [gigalith.component.interface :as component]))
    
(declare component-instance)

(mount/defstate component-instance
  :start (component/start! <args>)
  :stop  (component/stop! component-instance))

Helper Components

Helper components should be mounted if they have an init! method. This allows for using environment variables for init!.

(ns example
  (:require
    [mount.core :as mount]
    [gigalith.component.interface :as component]))
    
(mount/defstate component-instance
  :start (component/init! <args>))

Domain Components

Domain components generally rely on mounted components through a dependency-injection pattern of passing the required components into their functions. Example:

(ns example
  (:require
    [mount.core :as mount]
    [gigalith.database.interface :as database]
    [gigalith.patient.interface :as patient]))
    
(declare db)

(mount/defstate db
  :start (database/start! <args>)
  :stop  (database/stop! db))
  
(patient/get-by-id db <patient-id>)

Creating New Components

New components need to be initialized via the polylith tool:

project-root> poly new component name:<component-name>

The component then needs to be added to the base deps.edn as an extra-dep for the :dev alias’ :extra-deps:

{:aliases {:dev {:extra-deps {org.clojure/clojure       {:mvn/version "1.11.1"}
                              gigalith.<component-name> {:local/root "components/<component-name>"}
                              ..
                              }}}}

As well as the :test alias’ :extra-paths:

{:aliases {:test {:extra-paths ["components/<component-name>/test"
                                ..
                                ]}}}

interface methods should all use gigalith.malli.interface/defn and be as fully “specced out” as possible.
