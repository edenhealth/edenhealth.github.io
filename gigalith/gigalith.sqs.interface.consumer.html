gigalith.sqs.interface.consumer

SQS Consumer brick

Instantiate an sqs consumer reading and processing messages from an sqs queue.

To mount:

(declare consumer)

(mount/defstate consumer
  :start! (start! <opts>)
  :stop!  (stop! consumer))

Opts: - access-key-id aws access key id - secret-access-key aws secret access key - region aws region - host (optional) sqs endpoint - use for localstack - queue queue name - process-message-fn fn to process events - service-name name of the service - for metrics

Event structure:

{
  :event :string
  :data  :map
}

consumer-spec

SQS consumer instance spec.

[:map [:done-channel :any]
      [:message-channel :any]]

start-opts-spec

[:map [:access-key-id :string]
      [:secret-access-key :string]
      [:region :string]
      [:host {:optional true} :string]
      [:queue :string]
      [:process-message-fn :any]
      [:service-name :string]]

start!

(start! opts)

Starts an SQS consumer for queue using process-message-fn for processing messages.

Messages should be in the format:

{
  :event :string
  :data  :map
}

Arg Schemas:

[(opts :- start-opts-spec)]

Returns:

[:map [:done-channel :any]
      [:message-channel :any]]

stop!

(stop! consumer)

Stops an SQS consumer.

Arg Schemas:

[(consumer :- consumer-spec)]

Returns:

:nil
